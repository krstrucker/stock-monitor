# 클라우드 서버 배포 가이드

## ⚠️ 중요: 로컬 컴퓨터의 한계

**로컬 컴퓨터에서는:**
- ❌ 컴퓨터가 꺼져 있으면 아무것도 실행할 수 없습니다
- ❌ 컴퓨터를 자동으로 켜는 기능은 없습니다

**해결 방법: 클라우드 서버 사용**
- ✅ 24시간 항상 켜져 있음
- ✅ 컴퓨터가 꺼져 있어도 작동
- ✅ 인터넷만 있으면 어디서나 접속 가능

## 추천 클라우드 서비스

### 1. Railway (가장 추천) ⭐
- **장점**: 매우 간단, 무료 플랜 제공, 자동 배포
- **무료**: 월 5달러 크레딧 (충분함)
- **URL**: https://railway.app

### 2. Render
- **장점**: 무료 플랜 제공, 간단한 설정
- **무료**: 무료 플랜 있음 (약간 느림)
- **URL**: https://render.com

### 3. PythonAnywhere
- **장점**: Python 전용, 무료 플랜 제공
- **무료**: 무료 플랜 있음
- **URL**: https://www.pythonanywhere.com

### 4. Heroku
- **장점**: 널리 사용됨, 안정적
- **단점**: 무료 플랜 없음 (유료만)
- **URL**: https://www.heroku.com

## Railway 배포 방법 (가장 간단)

### 1단계: Railway 계정 생성
1. https://railway.app 접속
2. GitHub로 로그인 (또는 이메일)
3. "New Project" 클릭

### 2단계: 프로젝트 배포
1. "Deploy from GitHub repo" 선택
2. GitHub 저장소 연결
3. 자동으로 배포 시작

### 3단계: 환경 변수 설정
Railway 대시보드에서 환경 변수 추가:
```
TELEGRAM_BOT_TOKEN=your_token
TELEGRAM_CHAT_ID=your_chat_id
MONITOR_SYMBOL_COUNT=0
MONITOR_WORKERS=20
MONITOR_TIMEFRAME=short_swing
```

### 4단계: 완료
- 자동으로 URL 생성됨
- 24시간 항상 켜져 있음
- 컴퓨터가 꺼져 있어도 작동

## Render 배포 방법

### 1단계: Render 계정 생성
1. https://render.com 접속
2. GitHub로 로그인
3. "New +" → "Web Service" 선택

### 2단계: 저장소 연결
1. GitHub 저장소 선택
2. 설정:
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python server.py`
   - Environment: Python 3

### 3단계: 환경 변수 설정
Render 대시보드에서 환경 변수 추가

### 4단계: 완료
- 무료 플랜: 약간 느릴 수 있음
- 유료 플랜: 더 빠름

## PythonAnywhere 배포 방법

### 1단계: 계정 생성
1. https://www.pythonanywhere.com 접속
2. 무료 계정 생성

### 2단계: 파일 업로드
1. Files 탭에서 파일 업로드
2. 또는 GitHub에서 클론

### 3단계: Web App 생성
1. Web 탭 → "Add a new web app"
2. Python 3.x 선택
3. Flask 선택
4. 설정 완료

### 4단계: 스케줄러 설정
1. Tasks 탭에서 스케줄 작업 추가
2. 매일 22:30, 02:30에 실행
